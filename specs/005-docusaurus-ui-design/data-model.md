# Data Model: Docusaurus UI Configuration

**Date**: 2025-12-03
**Feature**: `005-docusaurus-ui-design`

## Overview

For this feature, a traditional data model (like a database schema) is not applicable, as the focus is on UI customization and configuration of a static site generator (Docusaurus). The "data" here refers to the structure of configuration files and CSS definitions.

This document outlines the relevant structural elements of the Docusaurus configuration files that will be modified to achieve the desired UI design.

## Configuration Structure

### 1. `docusaurus.config.ts`

This is the primary configuration file for the Docusaurus site. Key sections relevant to this feature include:

*   **`themeConfig.navbar.items`**: Defines the structure and content of the top navigation bar. This will be updated to include an "Introduction" link and a dynamically generated "Modules" dropdown.
    *   **Navbar Item (Introduction)**: `{ type: 'docSidebar', sidebarId: 'tutorialSidebar', position: 'left', label: 'Introduction', to: '/docs/intro' }`
    *   **Navbar Item (Modules Dropdown)**: This will be a custom component that dynamically reads module information.
*   **`themeConfig.footer.links`**: Defines the structure and content of the site footer. This will be updated to include direct links to Module 1, 2, 3, 4, and general chapter links.
    *   **Footer Link Group (`Book`)**: Will contain `items` for 'Introduction', 'Module 1 - ROS 2', 'Module 2 - Digital Twin', 'Module 3 - AI-Robot Brain', and 'Module 4 - VLA'.
*   **`themeConfig.colorMode`**: `respectPrefersColorScheme: true` is already set, ensuring basic theme mode handling.

### 2. `src/css/custom.css`

This file is used for applying custom CSS styles that override or extend Docusaurus's default Infima theme.

*   **Color Scheme Definition**: The `:root` selector will be used to define Infima CSS variables for a blue and white color scheme. This includes:
    *   `--ifm-color-primary` and its variations.
    *   `--ifm-color-white`, `--ifm-color-gray-X00`.
    *   Font, background, link, navbar, footer, and sidebar specific colors (`--ifm-font-color-base`, `--ifm-background-color`, `--ifm-navbar-background-color`, `--ifm-footer-background-color`, etc.).
*   **Mobile Responsiveness**: Targeted media queries (`@media (min-width: ...)` will be used here for any necessary fine-tuning of responsive behavior beyond Docusaurus defaults.

### 3. `sidebars.ts`

This file defines the structure of the documentation sidebar.

*   **`tutorialSidebar`**: Configured as `[{type: 'autogenerated', dirName: '.'}]`, meaning it automatically generates the sidebar based on the `docs` folder structure and `_category_.json` files. This will remain unchanged, as UI customization is handled via `_category_.json` files in `docs/moduleX`.

## Entities (Conceptual)

*   **Theme Variables**: CSS variables (Infima) defined in `custom.css`.
*   **Navbar Item**: An entry in the top navigation bar.
*   **Footer Link**: An entry in the bottom navigation.
*   **Module Category**: Represented by `_category_.json` files which control sidebar labels and positions.
